[system recommendation / dense + wild]

One grammar. Many states.

This visual compresses the favorite dense/wild concepts into a cohesive prompt system: regular shell remains the thin trace arrow, activated agent mode uses a bracketed operator frame, and the wild variants appear only when a state deserves it: execution, memory, route, lease, verification, or failure.

Shared grammar

arrow› remains the input anchor everywhere.
agent◈ is the activated-agent sigil.
bracket[hermes] identity + telemetry line.
frameBox-draw appears in agent mode, not shell mode.

› cargo test
Thin Trace Arrow

Your regular prompt. No brackets, no telemetry. It stays beautiful and command-focused.

Enter executes shell commands.
02 / agent defaultoperator-frame
╭─[hermes]──ctx:73%──tok:141k/192k
│ repo: anvil · tools: gated · evidence: on
╰─◈ › make this prompt cooler
Activated Agent Prompt

The best default: dense enough to feel like an agent cockpit, restrained enough to use constantly.

Enter sends instruction to Hermes/agent, not directly to shell.

│ write one file · tests required · no publish
╰─◈ ⏱ › approve bounded edit
Bounded Lease

Best for preventing takeover anxiety: exact scope, expiry, and allowed action.

Agent can act only inside the lease.

Takeover Guard

The prompt makes the boundary visible: agent can ask, but cannot silently act.

Human remains operator of record.
