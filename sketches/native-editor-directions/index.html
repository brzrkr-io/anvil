Anvil native editor rescue / visual companion

Three plausible native editor directions before more code ships

Research synthesis: current code already has a native buffer renderer, gutter, syntax, diagnostics, git marks, panes, explorer and outline stubs; what is missing is product legibility. These sketches keep the Ember palette, thin technical panels, monospace density, and guided next-action surfaces, while making file-open/editor state unmistakable.
