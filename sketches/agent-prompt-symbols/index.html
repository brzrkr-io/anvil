[new pass: symbols, shapes, and prompt grammar]

Stop using the same agent sigil.

Most agent prompts converge on diamonds, brackets, sparkles, and boxed telemetry because they are easy monospace tokens: visible, symmetric, and terminal-safe. This board tries fresher families: cursor hooks, apertures, chevrons, sockets, capsules, circuit nodes, basin marks, scanlines, and route ticks. The regular shell prompt remains separate; these are activated-agent-mode prompts only.

Why agents all look same

They all need the same jobs: distinguish mode, fit monospace, survive fonts, and show state fast. So everyone reaches for ◆ ◈ ✦ [] ▰.

Freshness should come from a repeatable grammar, not random Unicode soup.

Recommended Family

Pick one family, not random symbols: Hook for active agent, Basin for Anvil identity, Socket for systems/internal mode.

This feels fresher than diamonds while still making sense.

Best fresh symbol families

1. Hook / pulse:       ⌁
   Agent is attached to the session. Great default.

2. Basin half:         ◒
   Anvil-native identity. Great branded prompt.

3. Socket / endpoint:  ⊚
   Local agent process/control-plane feel.

4. Trace node:         ⦿
   Provenance/evidence/workflow feel.

5. Double prompt:      ››
   Most cohesive with shell, least flashy.

Why avoid common symbols

Overused agent glyphs:
  ◈ ◆ ✦ ✧ ✨ [] ▰

They work because they are symmetric and readable,
but they now feel like generic AI terminal cosplay.

Better rule:
  use a symbol that explains the agent relation:
  attached, routed, socketed, sealed, executing, verified.
