What shape should the Caldera prompt be?

All four carry the same info — current directory, git branch + dirty state, last exit status, command duration, time. Only the structure differs. Pick the direction that feels right; we'll tune colors, glyphs and which segments appear next. Click one (or tell me in the terminal).

caldera-console main ● ❯ zig build test caldera-console main ● ❯

A · Minimal single-line

Directory + branch + glyph, nothing else. Fastest to scan, lowest noise — the most "regular." Extra info (duration, exit code) shows only when it matters.

~/projects/caldera/caldera-console ⎇ main ●2 ✓ 1.2s 14:32 ❯

B · Rich single-line

Everything on one line, in segments. Most information density — but the line you type at sits far to the right after a long path.

~/projects/caldera/caldera-console ⎇ main ●2 ✓ 1.2s 14:32 ❯

C · Two-line

Context on top, a short clean line to type at below. Information never crowds what you write — the popular power-user setup.

╭ ~/projects/caldera/caldera-console ⎇ main ✓ 0.4s ╰❯ git status nothing to commit, working tree clean ╭ ~/projects/caldera/caldera-console ⎇ main ●2 ✓ 1.2s 14:32 ╰❯

D · Two-line, framed blocks

Like C, but a light bracket joins each prompt to its command — scrollback reads as distinct command blocks. Structure without becoming a TUI.
