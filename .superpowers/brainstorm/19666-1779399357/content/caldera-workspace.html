Caldera Workspace — Zed's chassis, a control plane's content

Zed gives the structure: a fast, modular workspace — docks at the edges, pluggable panels, everything toggleable. Caldera fills it with what an IDE doesn't have: observability. The panels aren't a file tree — they're live surfaces watching your repo, cluster, deploys, checks, agents. The prompt finally goes minimal because the control plane lives in the docks.

Modular all the way down — and built to be extended

Docks

Left · Right · Bottom edge containers. Each holds panels, toggles with a key, remembers its size. The Zed chassis.

Panels

Observability surfaces: Pulse, Cluster, Logs, Deploys, Checks, Agents — later Files & AI. Each is a web surface on the M3 bridge; a plugin can register a new one.

Modules

The live units inside a panel — repo, deploy, cluster, CI, incidents. Pluggable; an integration adds one with an icon, a value, a refresh rule.

 control plane, not just an IDE  —  a panel can go deep: the Logs panel tails prod, the Cluster panel charts pod health & p99, Deploys shows a release timeline, Checks drills into a failing CI step. Visibility an editor never has.
