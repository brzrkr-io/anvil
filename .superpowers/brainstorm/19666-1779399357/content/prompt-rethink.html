Mechanism · info leaves the prompt

1 · Status strip

All telemetry moves to a thin persistent bar. The prompt itself becomes a single pure glyph — the input line is never cluttered again. Info is ambient, always glanceable.

❯ zig build test
210 passed · 1.4s
❯ git status
working tree clean
❯ caldera · main ●2 · 09:16
Mechanism · the unit is a block, not a line

2 · Command blocks

Each command + its output is one quiet card; Caldera tints them using the OSC 133 marks it already parses. The active block carries a mineral edge. Warp-like structure, but it still reads as a plain terminal.

caldera ❯ quiet — clean repo, nothing to say
caldera main ●3 ❯ working — dirty, so it shows
caldera main ✗127 · 1.2s ❯ attention — last command failed
Mechanism · the prompt content is dynamic

3 · Adaptive

No fixed template. The prompt shows only what's true right now and grows as needed — silent on a clean repo, surfacing dirty count, exit codes, durations exactly when they matter. Minimal becomes the default state, not a compromise.

Mechanism · telemetry locked to a column

4 · Ledger

Your input always starts at column 0; every prompt's context is right-aligned into the same vertical column. Scrollback gains a tidy data column you can scan straight down — like reading a ledger.

❯ git status  ·  clean

Mechanism · Caldera draws it, not the shell

5 · Live header

The context strip is rendered by Caldera itself, so it isn't frozen text — it updates live: a spinner while a command runs, git state refreshing as files change, smooth color. The shell prompt below stays a pure ❯. Leans hard into Caldera owning the renderer.
