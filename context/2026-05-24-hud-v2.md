# HUD v2 Spec

## 1. Behavior — Expand Outward

**Constant.** Add `HUD_WIDTH_PT: f64 = 280.0` in `main.rs`.

**Toggle implementation.**

Today the `kb.hud_toggle` path in `handle_cmd_chord` and `Action::HudToggle` both call `self.resize_all_tabs()` on `App`, which shrinks the terminal. Replace this sequence:

1. Add `window: Retained<NSWindow>` to `AppShell`. Clone from `appkit.window` in `main()` before building the shell.
2. Add `fn toggle_hud(&mut self)` on `AppShell`. It flips `self.app.hud_visible`, then calls `window.setContentSize(NSSize { width: new_w_pt, height: self.app.view_height_pt })` where `new_w_pt = self.app.view_width_pt + HUD_WIDTH_PT` when opening and `- HUD_WIDTH_PT` when closing. Do not call `resize_all_tabs()` here — AppKit fires `AppShell::resize()` which calls it.
3. In `AppShell::perform_key_equivalent`, intercept `kb.hud_toggle` before it reaches `app.handle_cmd_chord`.
4. Remove `test!(kb.hud_toggle, ...)` from `App::handle_cmd_chord`. Update the `Action::HudToggle` branch to delegate or remove if unreachable.

**Why PTY cols don't change.** When opening: window grows by `HUD_WIDTH_PT`. `hud_visible` is true. `inner_rect()` subtracts `hud_cols * cw` from the new wider window — terminal area stays the same pixel width. Closing reverses.

**Edge case — screen boundary.** If the display constrains the grow, AppKit silently clamps. The terminal still does not shrink because `inner_rect()` subtracts the HUD allocation regardless. The HUD may render visually narrower than `HUD_COLS * cw` but the terminal is protected. Acceptable for v1.

## 2. Theme Migration

Delete `glass_tones_for()`, `GlassTones` struct, and `TonedTheme` struct entirely. Delete the local `const` color arrays. Replace every use with `app_theme` (the `&Theme` already passed into `draw_right_hud`).

| Element | Old | New |
|---|---|---|
| HUD surface fill | `GlassTones.surface` hardcoded | `app_theme.panel` |
| HUD left hairline | `GlassTones.edge` | `app_theme.hairline` |
| Section header labels | `GlassTones.label` | `app_theme.text_subtle` |
| Body text | `GlassTones.foreground` | `app_theme.foreground` |
| Meta / parent path / dim | `GlassTones.meta` / `const ALLOY` | `app_theme.text_muted` |
| Fold / gutter dim | `const ALLOY [0x86,0x91,0x9a]` | `app_theme.alloy` |
| Branch glyph, SHA, ports, PRs | `const INFO_TEAL [0x3a,0x8a,0x9d]` | `app_theme.info` |
| Clean git / CI pass | `const VERIFIED` | `app_theme.verified` |
| Dirty / pending / attention dot | `const ATTENTION` | `app_theme.attention` |
| CI failure / findings | `const FAILURE` | `app_theme.failure` |
| Agent / automation activity | `const AGENT_VIOLET` | `app_theme.agent` |
| Prod env kube dot | `ATTENTION` | `app_theme.attention` |
| Staging env kube dot | `INFO_TEAL` | `app_theme.info` |
| Dev / no-signal env dot | `ALLOY` | `app_theme.alloy` |
| Section break hairlines | `tones.edge` | `app_theme.hairline` |

`surface_alpha` values (0.88 dark / 0.72 light) preserved; only the surface RGB moves to `app_theme.panel`.

## 3. Layout and Typography

### Section headers
Replace `draw_section_header`'s `─ LABEL ───────` form. New form: plain label text in `app_theme.text_subtle`, starting at `inner_col`, no leading dash, no trailing fill. After the label row, draw a 1px hairline at `app_theme.hairline` spanning from `start_col + 1` to `start_col + hud_cols - 1` (reuse `draw_section_rule` on `header_row + 1`). Remove the duplicate inter-section hairline that fires via `section_break`; one hairline per section (below the header).

### Row text
- Primary value: `app_theme.foreground`
- Secondary / meta: `app_theme.text_muted`
- Tertiary / very dim: `app_theme.text_subtle`
- No box-drawing characters in row content.

### Icons
- All status dots: `●` (U+25CF) only. One geometry.
- Approval rows: change `▸` to `●` in `app_theme.attention`.
- Running agent rows: change `◆` to `●` in `app_theme.agent`.
- Failure findings: keep `✗` (U+2717) in `app_theme.failure`.
- Branch glyph: keep Nerd Font `\u{e0a0}`.
- No emoji anywhere.

### Spacing
- Left content pad: `inner_col = start_col + 2`. Keep.
- Row spacing within a section: one cell height per row. Keep.
- Section spacing: one blank row containing the hairline.

### Right-aligned values
Timestamps, mem GB, load number: draw from `max_col` leftward. Add `draw_text_right(col_right, row, text, color)` helper: `start = col_right.saturating_sub(text.chars().count())`, then `draw_text`.

### SYSTEM row
`mem ▄▅▆▆▃▁ N/N GB · load X.XX` — gauge + numbers in `app_theme.foreground`, labels `mem` / `load` in `app_theme.text_muted`. Right-align `load X.XX` to `max_col`.
