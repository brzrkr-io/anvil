## Durable lesson: main.zig integration diverged from the M3 plan

The M3 plan spec was written before M2 merged. M2 restructured `src/main.zig`
significantly: `App` gained a `TabManager` (`tabs`), `onKeyDown` grew a
structured `mods.command` branch with `handleTabKey`, `barRows()` split into
`topBarRows()` + `bottomBarRows()`, and several fields were renamed or moved.

When Task 7 wired M3 into `main.zig`, the plan's code snippets did not apply
directly — the integration had to be adapted to the post-M2 structure:
- ⌘K is intercepted inside the existing `mods.command` branch, after
  `handleTabKey`, not in a separate top-level guard.
- `webview.zig` was first truly compiled at Task 7 (not Task 6); two
  `std.fmt.allocPrintZ` call sites were corrected to
  `std.fmt.allocPrintSentinel(..., 0)` (Zig 0.16).

**Lesson**: spec code snippets for `main.zig` integration should be treated as
pseudocode when the file has changed between spec authoring and implementation.
Read the current `main.zig` before applying any plan snippet.
