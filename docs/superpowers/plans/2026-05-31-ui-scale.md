# Unified UI Scale (DPI-correct + live zoom) Implementation Plan

> **For agentic workers:** small verified commits, TDD the pure parts, `--dump` after every render-affecting change.

**Goal:** Anvil sizes itself correctly on any display (readable on a 4K out of the box) and offers one live knob (`Cmd =`/`-`/`0`) that scales font + chrome together, persisted.

**Architecture:** One scalar `s = ui_scale × backingScaleFactor` governs all device-px sizes.
- Font: `anvil_atlas_params.pt_size = font_size × ui_scale` (logical). Shim rasterizes at `pt_size × gRenderScale` where `gRenderScale = backingScaleFactor`. ⇒ glyph px = `font_size × s`.
- Chrome: `chrome.applyScale(s)` multiplies a base table (= current consts ÷ 2) by `s`.
- Backward-compatible: today's Retina is `backing=2, ui=1 ⇒ s=2`, which reproduces the current constants exactly. Headless `--dump` has no window ⇒ `backing` defaults 2.0 ⇒ golden renders unchanged.

**DPI-auto default:** when `backingScaleFactor < 2` and the user has not pinned `ui_scale` in config, default `ui_scale = 2.0` (makes a native-res 4K match the Retina physical size). Otherwise `ui_scale = 1.0`.

**Tech Stack:** Zig 0.16, AppKit/CoreText/Metal (Obj-C shim). Compiler `.zig/zig`.

- [ ] **Step 2: app — ui_scale + recompute path.**
```zig
var g_backing: f32 = 2.0;
var ui_scale: f32 = 1.0;
extern fn anvil_rebuild_atlas() callconv(.c) void;

export fn anvil_set_backing_scale(b: f32) callconv(.c) void {
    g_backing = if (b > 0) b else 2.0;
    applyUiScale();

    chrome.applyScale(ui_scale * g_backing);
    // bar_h-derived renderer pads recompute here (see Task 1 step 3)
}
```

  - `applyUiScale()` must run before the first atlas build: ensure `anvil_set_backing_scale` (called pre-buildAtlas) triggers it. The very first `anvil_atlas_params` then already reflects ui_scale + chrome is scaled.

## Self-review notes
- One scalar `s = ui_scale × backing` drives font + chrome ⇒ they never desync.
- `s=2` reproduces today exactly ⇒ Retina + golden `--dump` tests unaffected.
- Risk: `const bar_h`→runtime (Task 1 step 3) touches the renderer initializer; verify cell pads still correct after first `relayout`.
- Risk: `anvil_rebuild_atlas` re-rasterizes mid-session — ensure prewarm/atlas eviction state is reset inside `buildAtlas` (it allocates a fresh `gAtlas`), and that `anvil_set_metrics` fires so `cell_w/h` update before `relayout`.
